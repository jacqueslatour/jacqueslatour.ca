About TrustDID

A self-sovereign document signing and verification system built on open standards.

What Is TrustDID?

TrustDID is a document signing and verification system that uses Decentralized Identifiers (DIDs) — a W3C open standard — to establish verifiable digital identity without relying on centralized certificate authorities or blockchain networks.

With TrustDID, any organization can sign documents using cryptographic keys tied to their domain. Anyone can verify those signatures by resolving the signer's DID document — a publicly accessible JSON file hosted on the signer's own web server.

How Is This Different?

Traditional digital signatures depend on certificate authorities — trusted third parties that can be compromised, revoked, or subject to jurisdiction. TrustDID replaces that dependency with a self-sovereign model where your identity is your domain, your keys are your own, and verification requires no intermediary.

No Blockchain

Uses DNS and HTTPS infrastructure that already exists. No tokens, no consensus, no fees.

No Cloud Lock-in

Keys live on your filesystem. Sign documents offline. Host your DID on any web server.

Post-Quantum Ready

Every signature includes ML-DSA-44 alongside Ed25519, protecting documents against future quantum threats.

Privacy First

Verification happens locally. No document content is ever uploaded. The browser extension only sends the manifest URL for advanced verification.

What Is did:web?

did:web is a DID method that maps decentralized identifiers to standard HTTPS URLs. A DID like did:web:jacqueslatour.ca resolves to https://jacqueslatour.ca/.well-known/did.json — a JSON document containing the signer's public keys.

This means anyone can look up a signer's public keys simply by fetching a URL. No special infrastructure, no blockchain queries, no proprietary protocols. Just the web as it already works.

The Three-Layer Trust Model

TrustDID uses layered assurance levels so relying parties can choose how much verification they need:

  1. Standard — Cryptographic signature verified against the signer's DID document. Proves the signer holds the private key.
  2. High — Adds DNSSEC verification. A TLSA record in DNS anchors the signer's public key to their domain with cryptographic proof, independent of the web server.
  3. Maximum — Adds Trust Registry verification. The signer is registered in a trust ecosystem, providing organizational identity assurance beyond cryptographic proof.

This site operates at Maximum Assurance — every page is signed by a DID with both DNSSEC anchoring and Trust Registry authorization.

What Can Be Signed?

TrustDID can sign any file: HTML pages, PDFs, Word documents, images, spreadsheets, JSON data, source code — anything with a file path. The signing tool computes a SHA-256 hash of the file and creates a detached .vrfy manifest alongside it.

For websites, the trustdid_sign_dir tool batch-signs every file in a directory tree, making it trivial to publish a fully verified website like this one.

Open Source

TrustDID is open source. The signing tools, verification library, and browser extensions are available for review and contribution.
